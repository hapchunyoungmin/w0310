객체(코딩-object) vs 개체(디자인-object)

프로그래밍에서 객체는 프로그램을 구성하는 단위로, 개체는 관계형 
데이터베이스에서 표현하는 유형이나 무형의 실체를 의미합니다.  

객체
1.프로그램 설계 단계에서 최초로 생각해야 할 부분 
2.객체지향 프로그래밍(OOP)에서 모듈을 객체 단위로 하여 작성하기 
 쉽도록 함 
3.객체간의 인터페이스와 상속 기능을 통하여 객체 단위를 효율적으로
 재사용 할 수 있는 체계를 제공함 
4.클래스에서 정의한 것을 토대로 메모리에 할당된 것으로 프로그램
에서 사용되는 데이터 또는 식별자에 의해 참조되는 공간을 의미함 

개체
1.관계형 데이터베이스에서 표현하려는 유형, 무형의 실체로써 서로
 구별되는 것을 의미함 
2.하나의 개체는 하나 이상의 속성(attribute)으로 구성되고 각 속성은
 그 개체의 특성이나 상태를 설명함 
3.기본적으로 청사진에 따라 구성 및 할당된 메모리 블록임 
4.프로그램에서 동일한 클래스의 많은 개체를 만들 수 있음 
5.개체를 인스턴스라고도 하며, 명명된 변수나 배열 또는 컬렉션에 저장할 수 있음

HTML(Hypertext Markup Language)은 웹 페이지의 구조와 내용을
정의하는 언어로, 웹사이트의 기본 구성 요소입니다. 
HTML은 프로그래밍 언어가 아닙니다. 


URL(Uniform Resource Locator)은 인터넷에서 특정 리소스의 위치를 
나타내는 주소입니다. 웹 주소, 링크, 파일식별자, 유일자원지시기 등
으로도 불립니다. 

URL의 구성 요소 프로토콜(예: http 또는 https), 호스트 이름
(도메인 이름), 경로(리소스의 특정 위치). 

URL의 사용법
 1.URL을 사용하면 웹페이지, 이미지, 비디오, 오디오 파일, 문서 등을 찾을
 수 있습니다. 
 2.URL을 사용하면 온라인에서 어떤 항목이 어디에 있는지 식별하여 
 이를 보거나 다운로드할 수 있습니다. 
 3.URL은 웹에 게시된 리소스를 검색하기 위해 브라우저에서 사용하는 
 메커니즘입니다. 

# HTML 인용문

1. <blockquote cite=”URL”>: 긴 인용문
2. <q cite=”URL”>: 짧은 인용문
3. <abbr title=”의미”>: 약어, 두문자어
4. <address>: 문서 또는 기사의 작성자/소유자에 대한 연락처 정보
5. <cite>: 작품 제목

# HTML 컴퓨터 코드

1. <code>: 컴퓨터 코드 조각
2. <kbd>: 키보드 입력
3. <samp>: 컴퓨터 프로그램의 샘플 출력
4. <var>: 프로그래밍 또는 수학적 표현에서 변수

정보 구조 설계: 대중소로 이어지는 정보의 계층 구조를 만드는 과정이다. 
정보의 중요도와 연관성을 기반으로 구조적으로 설계한다. 내비게이션
설계: 사용자가 원하는 정보를 탐색할 수 있도록 하는 네비게이션을
설계하는 단계이다.

웹 내비게이션은 월드 와이드 웹의 정보자원 네트워크를 탐색하는 
과정을 의미합니다. 웹 사이트의 구조를 탐색하고 원하는 페이지로]
이동하는 데 사용되는 인터페이스 요소를 웹 사이트 네비게이션이라고
도 합니다. 

웹 사이트 네비게이션의 특징 

1.웹 사이트의 정보 구조를 나타냅니다
2.사용자가 원하는 정보를 쉽게 찾을 수 있도록 도와줍니다
3.웹 사이트 상단이나 사이드바에 위치하며, 메뉴로 구성됩니다
  사용자 경험에 직접적인 영향을 미칩니다

웹 사이트 네비게이션의 역할  

1.웹 사이트의 다양한 섹션과 페이지를 구성하는 데 활용됩니다
2.웹 사이트의 전체 구조를 시각적으로 보여주는 링크 모음을 제공합니다
 관련 콘텐츠를 제공합니다
3. 검색 기능을 제공합니다
4. 사용자의 동선과 행동 패턴을 파악할 수 있습니다

웹 사이트 네비게이션의 디자인

1.사용자 경험을 최적화하고, 방문자가 원하는 정보를 신속하게 찾을 수
 있도록 하는 것이 중요합니다 
2.반응형 디자인을 고려하여 다양한 화면 크기에 대응할 수 있도록
 설계되어야 합니다

미디어 쿼리(Media Query)는 CSS의 일부로, 미디어 유형이나 장치의
 특성에 따라 스타일을 조건부로 적용하는 기능입니다. 뷰포트의 
해상도에 따라 CSS를 분기시키는 기법으로, 반응형 웹 디자인의 핵심
 기술입니다

반응형 웹 특징이 아닌 것은?
 1. 모바일 기기에 최적화된 웹사이트이다.
 2.반응형 웹은 동일한 URL을 통해 다양한 디바이스에서 화면 크기에 
 따라 레이아웃이 자동으로 조정되는 웹사이트이다. 미디어 쿼리와 
 유동적 그리드 등을 사용해 한 웹사이트가 데스크톱, 태블릿,
 모바일 등 다양한 화면 크기에 맞춰 콘텐츠를 조정한다.
 3. 다양한 화면 크기에 맞춰 레이아웃이 자동변경되는 웹사이트이다.
 4. 미디어 쿼리를 통해 다양한 CSS 스타일을 적용하는 웹사이트이다.
